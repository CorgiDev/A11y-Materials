Form Components

The following is a test form with:

  1. First Name text input field with an autocomplete attribute set to given-name.
  2. Last Name text input field with an autocomplete attribute set to family-name.
  3. A Date Picker Combobox comprised of an editable date input combobox that opens a dialog containing a calendar grid and buttons for navigating by month and year.
  4. A Radio Group comprised of multiple radio buttons in a fieldset.
  5. A Two-State Checkbox Group made out of divs and checkboxes where each checkbox can be checked or unchecked.
  6. A Mixed-State Checkbox Group built with a fieldset and checkboxes where each checkbox is able to be in one of 3 states (checked, unchecked, and partially checked).
  7. Select-Only Combobox that functions similar to an HTML select element.

Some of these form elements were sourced from the W3C WAI ARIA Authoring Practices Guide (APG) Opens in new window.

When formatting a form that uses radio buttons and checkboxes, it is important to remember that these elements serve different semantic purposes, and should not be used inetrchangeably. Radio Options are meant to be used when only one option is allowed to be selected. Checkboxes are meant to be used when multiple options are meant to be selectable. However, Checkboxes can also be toggled between more states.

You can learn more on the Radio Group Pattern page Opens in new window and the Checkbox Design Pattern pageOpens in new window of the W3C WAI APG.

Buttons versus Links

Buttons and links, like checkboxes and radio options, are not interchangeable even though people love to use them interchangeably.

Buttons are meant to trigger an action or event (i.e., open modals/dialog, submit forms, cancel an action, expand a component, etc.). Links are meant to navigate to a target resource internal or external (i.e., a link to a spot within a page, to a PDF file, to an external site, or to another page within the site.).

You can learn more on the Link Pattern page Opens in new window and the Button Pattern pageOpens in new window of the W3C WAI APG.
